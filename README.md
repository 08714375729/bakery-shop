# 面包店系统（总店 / 分店账号版）

特点：
- **前端：一个 `index.html`**（手机友好，自适应）
- **后端：Flask + SQLite**（提供 REST API）
- **业务流程：总店录送货 → 分店逐款填写烂面包/收回金额 → 分门店日报 → 导出PDF**
- **账号权限：总店管理员 / 分店账号**
- **语言切换：中文 / Español**
- **账号功能：登录后可自行修改密码**
- **面包资料：支持默认单价，分店没单独定价时自动使用默认单价**
- **面包分类：总店录单时可先按分类选面包，更快**

## 1) 启动
在项目目录执行：

```bash
pip install -r requirements.txt --break-system-packages
python app.py initdb
python app.py
```

默认端口：`5001`

打开：
- http://localhost:5001/

> 注意：手机访问时，不能用 localhost。请用运行服务器那台电脑的局域网IP：`http://电脑IP:5001/`

## 2) 跑通“每天一次”的最短步骤
1. 进入【基础资料】：
   - 新增门店
   - 新增SKU
   - 给“门店+SKU”新增价格（不然应收为0）
2. 进入【总店送货】：
   - 选择日期、分店
   - 添加今天送了哪些面包、各送多少、按什么单价
   - 保存送货单
3. 进入【分店填写】：
   - 选择日期、分店
   - 填今天收回多少金额
   - 填今天损失多少金额，并可写备注
   - 保存（同一天同一分店再次保存会覆盖更新）
4. 进入【分门店日报】：
   - 查看每个分店：几款、总数量、送货金额、收回金额、损失金额、差额
   - 查看“烂面包明细”：哪家店、哪一款、烂了多少、损失多少
   - 点击【导出PDF】可打印/另存为 PDF

## 3) 说明（MVP简化）
- 不做门店下单
- 由**总店**录送货，由**分店**录“收回金额 / 逐款烂面包数量”
- 日报差额口径：`送货金额 - 收回金额 - 损失金额`
- `导出PDF` 使用浏览器打印能力，点击后可“另存为 PDF”
- 分店损失金额口径：`烂掉数量 × 送货单价`
- 如果某个分店还没设置专属价格，系统会先用“面包资料”里的默认单价
- 面包资料里可新增分类，例如：`甜面包 / 吐司 / 蛋糕 / 季节款`

## 4) 默认账号
- 总店管理员：`admin`
- 总店密码：`admin123`
- 分店账号规则：`store_门店编码`
- 分店初始密码：`123456`

例子：
- 门店编码如果是 `A01`
- 那分店账号就是：`store_a01`

## 5) 权限说明
- **总店账号**：可新增门店、面包、价格，录送货，看全部日报
- **分店账号**：只能看到自己门店，只能填写本店当天收回和烂面包

## 6) 语言切换
- 页面右上角可切换：
  - `中文`
  - `Español`

## 7) 修改密码
- 登录后，页面右上角有 `修改密码`
- 分店账号和总店账号都可以自己修改密码
